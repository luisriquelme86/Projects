- Javascript y PHP pendientes
. Javascript y PHP son diferentes a java en muchos aspectos, desde como se concibieron hasta sus primeros usos. Por ejemplo, Javascript corre en navegadores
  y PHP en un servidor. Por lo general mucho tiempo trabajaron juntos
. La idea es comenzar del principio en ambos lenguajes pero trabajandolo en el el ambito web, a diferencia de Java que el trabajo al principio es orientado a consola. Los lenguajes son
  diferentes y pensados y orientados a diferentes cosas y objetivos
. PHP en un principio no tenia clases y Java si, no lo hacia ni mejor ni peor, estaba planteado para otra cosa
. El gran problema que tuve siempre con Javascript y PHP es que lo aprendi como Java, orientado a solamente usar las funciones basicas y estructuras basicas, arrays y esas cosas
  pero nunca aprendi la parte web de listeners, ni usando html y css basico ni haciendo peticiones ni nada de eso, jamas conecta un javascript basico con un PHP basico
. Por ejemplo, en Java es muy comun hacer una agenda. En Javascript, con solo JS no seria posible porque no guardas estados, necesitas PHP con un server para que vaya guardando estados. Ese es un
  gran problema por no darle el enfoque que merece o estaba orientado cada lenguaje

- Consejos de programacion
. Asi como en java al programar muchas veces agarro un papel y hago dibujos, con js deberia hacer lo mismo. Por ejemplo, ahora 
  estoy haciendo que siempre que se agregue un nuevo elemento al array quede ordenado, pero no se como. Asi que dibujo un array, 
  tablas para indices y contenido y pienso. Lo mismo deberia hacer llegado el caso con js, "dibujar" un tag html, "dibujar" el listener o
  lo que sea que le corresponda, evento, etc, hacer el dibujo y poner lo que espero
. En el caso de javascript y PHP, podria hacer un dibujo "partiendo" con una linea la hoja al medio para dibujar lo que espero de un lado
  y otro de cada lugar (server y host)
